Add a function (can be inside `run()` or module-level) that parses a date string from a `labelText` value:
```typescript
function extractDate(labelText: string): string | null {
  // labelText format: "普通車ＡＭは2026年06月26日"
  const m = labelText.match(/(\d{4})年(\d{2})月(\d{2})日/);
  if (!m) return null;
  return `${m[1]}${m[2]}${m[3]}`;  // YYYYMMDD
}
```

**Step 4 — Compute today's date string** (DET-05, D-05):

```typescript
const now = new Date();
const today = `${now.getFullYear()}${String(now.getMonth() + 1).padStart(2, '0')}${String(now.getDate()).padStart(2, '0')}`;
```

**Step 5 — Filter pipeline** (DET-04, DET-05, DET-06):
```typescript
const qualifying = allData.filter(rec => {
  // Must have 'enable' class
  if (!rec.classList.includes('enable')) return false;
  // Category filter
  if (SLOT_CATEGORY && !rec.labelText.startsWith(SLOT_CATEGORY)) return false;
  // Date extraction
  const date = extractDate(rec.labelText);
  if (!date) return false;
  // Past-date filter: skip dates before today
  if (date < today) return false;
  // Threshold filter: skip dates on or after ALERT_BEFORE (if set)
  if (ALERT_BEFORE && date >= ALERT_BEFORE) return false;
  return true;
});
```

**Step 6 — Deduplicate qualifying dates**:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env → filter logic | ALERT_BEFORE and SLOT_CATEGORY from .env; malformed ALERT_BEFORE (non-YYYYMMDD) would cause string comparison to produce wrong results |
| state.json → process | File is read on every run; corrupt JSON triggers the catch fallback silently |
| DOM labelText → date parser | Third-party page content; regex only extracts if format matches — returns null otherwise |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | output/state.json | accept | Local file, no network exposure; worst case is a false re-alert on next run if manually corrupted; catch fallback resets to safe defaults |
| T-02-05 | Tampering | ALERT_BEFORE env var | accept | String comparison (date >= ALERT_BEFORE) is safe for YYYYMMDD lexicographic ordering; document expected format in .env.example |
| T-02-06 | Spoofing | labelText from DOM | accept | Data used only for date parsing and logging; no code execution path from DOM content |
| T-02-07 | Denial of Service | state.json write failure | mitigate | fs.mkdirSync ensures output/ exists before write; any write error bubbles to master timeout catch → ERROR log + exit 0 |
| T-02-08 | Information Disclosure | console output | accept | Only slot dates (public booking availability) are logged; no credentials, PII, or session tokens in output |
</threat_model>
